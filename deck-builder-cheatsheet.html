The Workflow
01Research the actual content -- read the source, don't just format what's pasted
02Capture the moment -- audience, format, feeling
03Choose a style -- named catalog, URL, images/folder, or describe it
04Structure into a narrative arc with domain awareness
05Build from the starter template -- one HTML file, zero dependencies
06Open and iterate -- edit in place, don't regenerate
07Deploy to GitHub Pages with optional password gate

Workshop Pedagogy
  • Earn the tension before the solution
  • Vary visual layout every 3-4 slides
  • Audience does something every 5-7 min
  • One "screenshot this" cheat-sheet slide
  • Annotations are audience-facing, not notes
  • Exercise must produce a usable artifact
  • Virtual: chat first, breakouts of 2-3, auto-return
Core Principles
  • The deck is a tool, not a slideshow
  • Fewer slides, more interactivity
  • Match the moment, not the template
  • One file, zero build steps
  • Every slide must do something PPTX can't
  • Research the content, don't just format it
  • Contrast check before shipping (always)
